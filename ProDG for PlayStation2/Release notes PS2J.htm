概要 トップに戻る

  • 今回のフルツールチェーン・リリースには、以前のビルドツールのリリースには含まれていたLink Onceサポートも入っています。Link Onceを使うと、リリースビルド・コードを大きく膨らませてしまう重複したテンプレート・インスタンスを無くすことができます。できるだけ小さなELFを作るためのより詳しい情報は、ProDGのホワイトペーペー"ProDG for PlayStation 2最適化のアドバイス."をご覧ください。

    さらに、テンプレートを多用した大きなC++プロジェクトに見られたコンパイル負荷を調べ、その結果を基にコンパイラ実行イメージの性能を向上させました｡性能向上はコーディングスタイルによって変わりますが、当社のテストでは大きなC++プロジェクトのいくつかで大幅なコンパイル時間の短縮が見られています｡

  • iopfixup (v2.01)の新バージョンが含まれています。これはIOP 2.95.2 コンパイラ使用時に必要なものです。
  • デバッガの新機能. 新しい mixed source/disassembly ビューをご覧ください。
  • Target Manager は、新しく追加されたファイルサービス用コマンドをサポートします。
  • このリリースには、ProDG PLUS for PlayStation 2　ライセンスをもつ方にのみ有効な機能、例えばデバッガスクリプティング､を一部含みます。

今回行われた新機能追加と大幅ななバグ修正を以下に示します。ここに挙げた以外にもバグ修正が行われていますが数が多いので記載を省略しているものもあります。

EE コンパイラ

  • 機能削除: インストーラから GCC 2.95.2 と 2.9-991111 を削除しました。ウェブサイトには置いてあります。
  • 機能変更: コード生成機能を変更し、ディレイスロット内にオプコードに関係したFDIV機能ユニットが入らないようにしました｡これはハードウェア上の理由によるSony EE CPUの問題のためです。
  • 機能追加: #pragma mathwarn. 浮動小数点演算の警告設定のオン/オフを関数ごとに行なえるようにするためです。
  • 機能追加:リンク時に複製されたC++オブジェクトを除く為、コンパイラが"link once"セクション名を使用(例え：複製されたインライン及びテンプレートなど)。ただし-gフラグを使用の場合、"link once"は使用されません。
  • 機能追加: スイッチ -fforce-link-onceを追加しました。
  • 機能追加: スイッチ -fno-inline-debuggingを追加しました。インラインコードに対してソースデバッグ情報が生成されるのを防止します。
  • 機能追加: スイッチ -Wno-missing-enum-caseを追加しました。switch文の中にenumがない場合の警告を止めます。
  • 機能追加: C++の初期化されていないグローバルデータは.bssセクションを使うようになりました。
  • 機能追加: アラインメントされていないload/storeビルトイン呼び出し。使用例： int __builtin_getunaligned ( void* ptr ); void __builtin_setunaligned ( void* ptr, int value ); このふたつのコンパイラ関数は、そのアラインメントに関わらずintsをメモリに書き込み、読み込みするのを許可します。
  • 機能追加: スイッチ -fno-static-dtors (C++)を追加しました。スタティック・グローバル・デストラクタを呼び出すコードを生成しません。これはゲームのコードが決して終了せず、自身のスタティック・クラスインスタンスを廃棄する必要が絶対にない場合にコードサイズを減らすことができます。
  • 機能追加: スイッチ -fopt-stackを追加しました。64ビットか128ビットかのレジスタ保存を最適化します。
  • 機能追加: コンパイラファイルはバイナリモードで書き込まれるようにオープンされます。これでビルド時間が短縮されます｡
  • 機能修正: 不正に組み立てられたasm制約で共通のサブエクスプレッションの最適化が無限ループに入ってしまう問題。
  • 機能修正: スタブデバッグデータがwide-char型のために変更されました。
  • 機能修正: アラインされていないlong intをアクセスする際の誤ったモード。
  • 機能修正: 非常に大きなマイナスのenum値の精度。
  • 機能修正: テンプレート用にバインディングをセーブする時のコンパイラのホットスポット。
  • 機能修正: 'file function name'の形式がファイル名にランダムな数がつくように変更しました。これは、コンパイルモジュールで最初に見つかる関数が多くのコンパイルモジュール中に重複してしまうというC++コンストラクタの問題を解決します。

制限事項および警告 トップに戻る

宣言事項:

  • VSIでビルドされたDLLは、そのdspファイルが位置するディレクトリのコマンドラインから起動されたデバッガでのみデバッギングする事が可能です。
  • ee-objcopyは、-jスイッチをサポートしてません。
  • ee-gccは、GCC 2.9-991111を使用するようハードコード化されていましたが、このインストーラには含まれていません。その代わり、ee-gcc2953を提供しています。

警告:

  • このリリースに含まれるlibc.aには、新しいsnprintf関数が含まれています。これは、"Proview for PlayStationR2"内にある同じ名前の関数とコンフリクトを生じます。この問題の改善の為、今後のProViewリリースでこの関数を変更する予定です。

    C++使用時のリンカスクリプト変更:

    C++リンクが正常に行われるように、app.cmdを変更する必要があります。このapp.cmdファイルは通常\usr\local\sce\ee\lib\app.cmdディレクトリに位置しますが、ローカルコピーapp.cmdを使用してビルドしている場合があります。これらのapp.cmdファイルは変更する必要があります。変更方法は、次のURLにあるアーティクルよりご参照さい。
     "http://www.snsys.com/Support/Library/article.asp?id=1952"

    スクリプトに何の変更が必要か、以下のリストを参照してください。

    • Sony libs v2.6.0のapp.cmd
      • このバージョンのリンカスクリプトは既にアップデートされています。
    • Sony libs v 2.5.Xのapp.cmd
      • このバージョンのリンカスクリプトをご使用の場合、通常小データセクションに*(.sdata)とありますが、*(.sdata)の後に*(.gnu.linkonce.s*)を追加して下さい。
    • Sony libs v 2.4.Xのapp.cmd
      • このバージョンのリンカスクリプトをご使用の場合、下記のURLのドキュメントを参照しリンカスクリプトの変更を行ってください。
         "http://www.snsys.com/Support/Library/article.asp?id=1952"
