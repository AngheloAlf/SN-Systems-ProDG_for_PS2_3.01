Overview

  • This release uses a "link once" style of linking a C++ based game. "Link once" eliminates duplicate instances of inline methods, duplicate template instances and other duplicate C++ constructs. This considerably reduces the size of your ELF file. In addition, options are available in the compiler and linker to remove global destructors from your game. Global destructors will never be used as, typically, the game never exits. There have been improvements to the tools so that stripping of functions using the -strip-unused linker option is more efficient. In addition further stripping of unused part of your game is possible with this release by using the -strip-unused-data linker option. The -strip-unused-data option removes unused data as well as unused functions. For more information and tips for producing the smallest ELF possible, please see the white paper: Tips for optimising your code using ProDG for PlayStation®2.

The compiler has been modified to run more quickly with C++ compilations. The amount of improvement you will see will depend on the style of you C++. For tips on making further improvements to the build speed by improving the style of your C++, please see the white paper: Tips for optimising your code using ProDG for PlayStation®2.

  • A new version of iopfixup (v2.01) is included which is essential when using the IOP 2.95.2 compiler.

  • The debugger has several new features. Check out the new mixed source/disassembly view.
  • The Target Manager supports additional file serving commands.
  • This release describes some features eg. debugger scripting, which are only enabled if you hold a license for ProDG PLUS for PlayStation®2.

The following major features and significant bug fixes have been made. 

EE Compiler

  • REMOVED: GCC 2.95.2 and 2.9-991111 from installer. Still available on the website.
  • CHANGED: Code generation changed to ensure no FDIV functional unit related opcodes in delay slots. This is due to a Sony EE CPU caution regarding a hardware problem
  • ADDED: #pragma mathwarn. Allows toggling of the floating point math warning setting on a function by function basis.
  • ADDED: "link once" style section naming is now used by the compiler to eliminate various duplicate C++ objects at link time. For example duplicate methods declared inline and duplicate template instances. Note "link once" is not used when compiling with the debug generation command line flag "-g".
  • ADDED: Switch -fforce-link-once. Forces use of "link once" section naming when generating debug information.
  • ADDED: Switch -fno-inline-debugging. Prevents source debugging information being generated for inlined code. 
  • ADDED: Switch -Wno-missing-enum-case to suppress warnings about missing enum cases in switch statements.
  • ADDED: C++ non-initialised global data now uses .bss section.
  • ADDED: Unaligned load/store built in calls. Usage: int __builtin_getunaligned ( void* ptr ); void __builtin_setunaligned ( void* ptr, int value ); These two built in compiler functions allow the reading and writing of int(s) to memory regardless to their alignment.
  • ADDED: Switch -fno-static-dtors (C++). Prevents generation of code to call static global destructors. This reduces code size when game code never 'ends' and therefore never requires to destroy its static class instances.
  • ADDED: Switch -fopt-stack. Optimises 64 vs 128 bit register saving. 
  • ADDED: All compiler files now opened for writing in binary mode. This decreases build time.
  • FIXED: Badly constructed asm constraints causing common sub-expression optimisation pass to enter infinite loop.
  • FIXED: Stabs debug data changes for wide-char types.
  • FIXED: Use of wrong mode when accessing unaligned long int(s).
  • FIXED: Very large negative enum values losing precision.
  • FIXED: Hot-spot in compiler when saving bindings for templates.
  • FIXED: Changed 'file function name' format to filename plus random. This fixes C++ constructor problems when the first function found in a compilation module is duplicated in many compilation modules.

IOP Compiler and tools

  • ADDED: GCC 2.95.2 is now the default compiler set by the SN.INI generated by the installer. GCC 2.8.1 is still included and will remain your default if you omit to generate a new SN.INI. 
  • FIXED: iopfixup Hi16/Lo16 relocation problem, exhibited when using the 2.95.2 C/C++ compiler

Compatibility with Sony libraries back to top

This release has been tested with v2.6.0 of the Sony runtime libraries.

Please note that this release is not compatible with Sony's 'latest everything' since it contains the Sony Linux compiler 2.96 and tools. You only need to install the Sony runtime libraries (usually a zip file with name tlibs-2.6.0.tgz for example) not the 'latest everything' (zip file tools_libs-2.6.0.tgz). If you have installed the 'latest everything' under Win32, then this installation will overwrite portions of it.

Limitations & Warnings back to top

Limitation:

  • DLLs built with VSI can only be debugged if ps2dbg.exe is launched from the command line in the directory where the DLL's dsp file resides.
  • ee-objcopy does not support -j
  • ee-gcc is hard coded to use GCC 2.9-991111 which is no longer in this installer. However, we do include ee-gcc2953 which can be used instead.

Warning:

  • The libc.a in this release contains a new function snprintf. This will conflict with a function of the same name in "Proview for PlayStation®2". Proview has now been changed.

  • Required changes to linker script when using C++:

The app.cmd file may need updating before C++ linking will work correctly. The app.cmd file is usually found in your "\usr\local\sce\ee\lib\app.cmd" directory but you may have builds that use local copies. These too will need modifying. The document: "http://www.snsys.com/Support/Library/article.asp?id=1952" contains full instructions.

Please check what changes your script requires by looking at the list below.

app.cmd supplied with Sony libs version 2.6.0
---------------------------------------------
The linker script provided with Sony libs 2.6.0 has already been updated.

app.cmd supplied with Sony libs versions 2.5.x
---------------------------------------------
Linker Scripts from version 2.5.x of the Sony libs require that in the small data part of the script that usually contains:
*(.sdata)
you add:
*(.gnu.linkonce.s*)

app.cmd supplied with Sony libs version 2.4.x
--------------------------------------------
The linker script provided with versions 2.4.x of the Sony libs requires the changes outlined in the document:
"http://www.snsys.com/Support/Library/article.asp?id=1952"
